The one-paragraph version

Tinfoil lets companies and people use AI models without anyone — not even Tinfoil — being able to see the data they send in.

Today

Paste a patient record or proprietary code into ChatGPT and you are trusting a legal promise not to read or keep it.

With Tinfoil

That promise is replaced with a hardware-enforced, mathematical proof that the data is invisible to everyone — including Tinfoil's own engineers and the cloud provider.

Mental model: “Apple’s private AI cloud — but for any model and any company.”

The problem: encryption has a blind spot

Data is protected in transit and on disk — but to actually run the model, it must become readable plaintext in memory.

How the technology works

Three ideas:
enclave, attestation, transparency

Explained from zero. The punchline: you don’t trust Tinfoil — you trust silicon and math.

Idea 1 — the secure enclave (TEE)

Modern NVIDIA / AMD / Intel chips can create a “sealed room” inside the hardware. Memory in it is encrypted by the chip itself.

One cloud machine
Operating system — can’t see in Hypervisor — can’t see in Cloud staff / admin — can’t see in
Trusted Execution Environment
The AI model + your decrypted data live here. RAM is hardware-encrypted; the CPU↔GPU (NVIDIA H100/Blackwell) link is encrypted over PCIe. Plaintext exists only transiently, inside this boundary.

But: a sealed room is worthless if you can’t prove your data went into that room running honest code. → Idea 2.

Idea 2 — remote attestation

The hardware can prove, cryptographically, exactly what code is running inside the enclave.

1
At fabricationThe chip vendor (NVIDIA / AMD / Intel) burns a secret signing key permanently into the silicon.
2
At bootThe hardware takes a cryptographic fingerprint (a “measurement”) of exactly what firmware, kernel, and application binary is loaded.
3
It signs the fingerprintSigned with the burned-in key → an attestation report: tamper-proof proof of “this exact code is running, on genuine vendor silicon.”
4
You verifyAnyone can check that signature chains back to the manufacturer’s public root. Fake or modified code → different fingerprint → fails.

Binding the encryption keys to the enclave

Attestation proves the code. One more step ties your encrypted channel to that exact proven enclave.

The keys are cryptographically welded to the measured code. Talk to that key = talk to that exact enclave.

A man-in-the-middle would need a different key → not in a valid signed report → the client refuses to send data. No trusted middleman required.

Idea 3 — public transparency (no “trust us”)

How does your client know which fingerprint is the honest one? Public, tamper-proof logs.

Fetches the expected measurement (GitHub + Sigstore) → pulls the live enclave’s attestation → compares → only completes the encrypted connection, pinned to the attested key, if they match. Automatic. No proprietary attestation service.

The trust model in one slide

You DO trust

  • NVIDIA / AMD / Intel silicon & their root keys
  • Standard cryptography & math
  • Public transparency logs being append-only

You do NOT trust

  • Tinfoil (the company / its staff)
  • The cloud / GPU provider
  • Host operators & the network

So what: the security guarantee is reduced from “a company’s promise” to “chip vendors + math.” That is a genuinely different — and stronger — model. The honest caveat: you still must trust the silicon vendors’ secret processes.

The moat — and how durable

NOT defensible

The core tech. Enclaves & attestation are NVIDIA/AMD/Intel hardware features anyone can use. Tinfoil even open-sources its enclave code.

Actually defensible

  • Trust brand & verifiability UX
  • Developer-first, drop-in distribution
  • Speed & focus vs. distracted incumbents

Durability: moderate-to-fragile

It’s an execution-and-brand moat, not a technology moat. It holds only if they out-execute on trust, model breadth, and ease-of-use before a hyperscaler ships the same thing as a checkbox.

Overall assessment

A sharp, technically excellent company solving a real and growing problem, with a top-tier team — but in a structurally hard position.

The thesis is sound: as AI moves into regulated, high-stakes data, “trust us” stops being acceptable and cryptographic proof becomes valuable. Tinfoil is arguably the most credible developer-friendly, radically-transparent player in that space today.

The catch: the advantage is trust/brand/speed — not defensible tech — and the likely buyer of “good enough” privacy can get it contractually with better models. Realistic outcomes skew toward strategic acquisition or a solid niche. As an investment: team & thesis A-grade; push hard on moat durability and capital.
